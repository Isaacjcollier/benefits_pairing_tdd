What is Paired Programming?

Paired Programming

  • Agile software development technique
  • Always engaged in process of development
  • Driver and navigator

Driver

  • Hands on coding
  • Implementor
  • Syntax focused

Navigator

  • Architect
  • Second set of eyes at all times
  • Addressing future problems
  • Garbage pick up

Benefits of Pairing

  • Better code quality
  • Increased confidence in code deployment
  • Increased discipline and accountability
  • Constantly learning
  • Everyday is a code review
  • Prevent silo knowledge
